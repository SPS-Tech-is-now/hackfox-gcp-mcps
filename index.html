DEMO EN VIVO — DEMO 1
Knowledge MCP — Una pregunta que Google no puede responder fácil

La pregunta que nadie sabe responder de memoria

Una pregunta con tres capas técnicas que requiere documentación actual — ningún LLM de entrenamiento la responde bien porque Firebase AI Logic salió en 2025 y los safety thresholds cambian con cada versión.

Agente con Knowledge MCP activo — google-developer-knowledge
# Pregunta al Knowledge MCP:

Gemini 2.0 Flash en Firebase AI Logic —
¿cuándo uso inlineData vs fileData para fotos desde móvil?
¿Hay límite de tamaño?
Y si el safety filter bloquea fotos de infraestructura dañada
(banquetas rotas, postes caídos), ¿cómo ajusto los thresholds?

→ consulta Firebase AI Logic docs + Gemini API docs
→ cruza límites de inlineData (20 MB) vs fileData (Cloud Storage URL)
→ localiza la tabla de HarmBlockThreshold actualizada
DEMO 1 — Respuesta
Respuesta — con fuentes de la documentación oficial
inlineData → base64 en el request · límite 20 MB · ok para fotos móvil (≤ 5 MB típico)
fileData → gs:// URI de Cloud Storage · sin límite práctico · requiere bucket mismo proyecto

Safety config para contenido urbano / infraestructura:
HARM_CATEGORY_DANGEROUS_CONTENT → BLOCK_ONLY_HIGH
HARM_CATEGORY_VIOLENCE → BLOCK_ONLY_HIGH
(BLOCK_MEDIUM_AND_ABOVE bloquea fotos de daño urbano legítimo)

# Código generado desde la doc oficial — no de memoria de entrenamiento:
const result = await model.generateContent([
{ inlineData: { data: base64img, mimeType: 'image/jpeg' } },
'Clasifica el tipo de barrera urbana en esta imagen.'
], { safetySettings: [...] });

✓ Respuesta precisa · fuentes citadas · código listo para pegar
⚠ Sin MCP esto tarda 15-20 min y probablemente llega a la doc incorrecta
Por qué esta pregunta es el benchmark perfecto

Cualquier LLM de entrenamiento tiene la respuesta desactualizada — Firebase AI Logic se lanzó en 2025 y los thresholds de safety cambian con cada versión de la API. Knowledge MCP obtiene la respuesta correcta hoy.

Paso 1 — El equipo define el contrato (10 min al inicio del hackathon)
contracts/barriers.ts — la fuente única de verdad del equipo
// Diseñado juntos en 10 min · commiteado al repo · todos lo leen

export type BarrierType =
'broken_sidewalk' | 'missing_ramp' | 'blocked_path'
| 'broken_light' | 'pothole' | 'other';

export interface Barrier {
id: string; // Firestore doc ID
type: BarrierType;
severity: 'low' | 'medium' | 'high' | 'critical';
coords: { lat: number; lng: number };
photos: string[]; // gs:// URIs
status: 'open' | 'in_review' | 'resolved';
reportedBy: string; // userId
votes: number;
createdAt: Timestamp;
}

export interface CreateBarrierRequest {
type: BarrierType; severity: Barrier['severity'];
coords: Barrier['coords']; photoBase64?: string;
}

Q&A

¿Preguntas?

preguntas frecuentes — respuestas directas
Q ¿Cuánto cuesta todo esto?
A Firebase Studio: gratis (3 workspaces en preview). Claude Desktop: gratis. Créditos GCP del evento cubren las APIs. Maps: ~$5 USD sobran.

Q ¿Funciona con Gemini en lugar de Claude?
A MCP es protocolo abierto — Gemini, GPT-4, Claude. Para este stack de Google + Thinking mode, Sonnet 4.6 es la combinación más fuerte que encontré.

Q ¿Tenemos que usar Firebase Studio?
A No. VS Code + Claude Code Extension o Cursor funcionan bien para backend. Ventaja de Firebase Studio: plugins de Google ya son los MCPs, cero configuración.

Q ¿Y si se cae el WiFi?
A Claude Code CLI funciona offline para editar y razonar. Regla: primeras 4 horas con WiFi → generar todo lo posible. Después viene el refinamiento.

Q ¿Los MCPs pueden escribir a nuestra base de datos?
A Sí. Firebase MCP tiene R/W. Por eso los roles IAM importan — dar al service account solo los permisos mínimos. En el repo hay una guía de roles por operación.
